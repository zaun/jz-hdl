# Test Plan: 6.4 CLOCKS Block (including 6.4.1 CLOCK_GEN)

**Specification Reference:** Section 6.4 and 6.4.1 of jz-hdl-specification.md

## 1. Objective

Verify CLOCKS block (period, edge properties), clock source validation (must be IN_PIN or CLOCK_GEN output, not both), CLOCK_GEN types (PLL, DLL, CLKDIV, OSC, BUF), CONFIG parameter validation against chip data, and clock uniqueness.

## 2. Instrumentation Strategy

- **Span: `sem.clock_validate`** — attributes: `clock_name`, `period`, `edge`, `source`.
- **Span: `sem.clock_gen`** — attributes: `gen_type`, `input_clock`, `output_clocks`, `config_params`.

## 3. Test Scenarios

### 3.1 Happy Path
1. IN_PIN clock with period: `sys_clk = { period=10 };`
2. CLOCK_GEN PLL with multiple outputs
3. CLOCK_GEN CLKDIV
4. CLOCK_GEN OSC (no input)
5. CLOCK_GEN BUF
6. Falling edge clock
7. Clock with no optional config: `clk;`

### 3.2 Boundary/Edge Cases
1. Very fast clock: period=0.5 (2 GHz)
2. Very slow clock: period=1000000
3. CLOCK_GEN with optional CONFIG params omitted

### 3.3 Negative Testing
1. Clock in both IN_PIN and CLOCK_GEN — Error
2. CLOCK_GEN output with period specified — Error
3. IN_PIN clock without period — Error (for external clocks)
4. Duplicate clock name
5. Non-positive period
6. Invalid gen_type
7. Unknown CONFIG parameter for chip
8. CLOCK_GEN input from same block — Error (self-reference)
