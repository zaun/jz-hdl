**Role:** You are a Senior SDET specializing in C99 systems and OpenTelemetry instrumentation.

**Context:** 
I am building a C99 application managed with CMake. I have two reference documents:
1. **The Language Specification:** This defines the functional requirements, logic, and expected behavior of each module.
2. **The Dependencies Document:** This defines external libraries (e.g., OpenTelemetry, CURL, specific hardware drivers) and their integration points.

**Task:**
Create a comprehensive **Test Plan** for every individual subsection identified in the Language Specification.

### Prerequisite:
Delete existing tests pipeline/test_*.md

### Test Plan:
1.  **Objective:** What specific functionality/requirement is being verified?
2.  **Instrumentation Strategy:** How will we use OpenTelemetry spans or custom hooks to verify internal execution flow (ensuring "code coverage" visibility)?
3.  **Test Scenarios:** 
    *   **Happy Path:** Standard inputs that should succeed.
    *   **Boundary/Edge Cases:** Minimum/maximum values, empty strings, null pointers.
    *   **Negative Testing:** Specifically how the module should handle failures from its dependencies (e.g., OTel exporter is unreachable, a dependency returns a non-zero exit code).
4.  **Input/Output Matrix:** 
    *   A list of input variables and their expected states.
    *   The expected side effects (logs, OTel spans, or return codes).
5.  **Integration Points:** Identify which dependencies from the **Dependencies Document** are touched by this subsection and how to mock or stub them for isolated unit testing.
6.  **Rules Matrix:**
    *   A list of language rules to be tested.
    *   A list of language rules that are missing.

### Formatting:
Format the output in Markdown. Use headers for each subsection and tables for the Input/Output matrices.

### Documents to process:
compiler/src/rules.c
specification/jz-hdl-specification.md
pipeline/dependancies.md

### Output Location:
File name should be lowercase and use underscores for spaces.
pipeline/test_<section>_<subsection>-<plan_name>.md
pipeline/test_<section>_<subsection>_<subsubsection>-<plan_name>.md
