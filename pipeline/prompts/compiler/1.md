Act as a software architect and static analysis tool. Your goal is to generate a comprehensive dependency graph of the current codebase to facilitate automated test creation. 

Analyze the source code and create a Markdown file containing a Mermaid.js 'graph TD' (Top-Down) diagram.

### Prerequisite:
Delete existing pipeline/dependancies.md

### Requirements:
1. **Scope**: Identify all major components, including classes, modules, and functions.
2. **Relationships**: Map how these components interact (e.g., "Component A calls Component B", "Class X inherits from Class Y").
3. **Categorization**: Use Mermaid 'subgraph' syntax to group files by their directory/layer (e.g., /ast, /ir, /lsp).
4. **Dependency Types**: 
    - Use solid lines (-->) for internal code dependencies.
    - Use dotted lines (-.->) for external API calls or third-party library integrations.
5. **Chart**: Use flowchart LR with elk layout config. Do NOT use graph TD.
6. **Critical Path Metadata**: Include the following sections below the graph
    - Side Effects & Mocking Requirements
        * Table must include: Module, Side Effects, Hardcoded Dependencies Requiring Mocks
    - Leaf Nodes (no internal dependencies — unit test first)
        * Table must include: File, Role, Test Strategy
    - Root Nodes (highest dependency count — integration test)
        * Table must include: File, Role, Test Strategy
    - High-Value Intermediate Nodes (moderate deps — focused integration tests)
        * Table must include: File, Role, Test Strategy

### Output Format:
- Start with a # Dependency Graph title.
- Provide the Mermaid code block.
- Provide a "Critical Path" section identifying which files are 'leaf' nodes (no dependencies) vs 'root' nodes (high dependency count), to help prioritize unit vs integration tests.

Search through the files starting from the main entry point to ensure no critical links are missed.

### Output Location:
Store the output as pipeline/dependancies.md
